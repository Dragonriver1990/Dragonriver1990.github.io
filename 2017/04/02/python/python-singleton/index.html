发表于 2017-04-02 | 分类于 Python |

简介

方法一：装饰器实现

实际上就是利用装饰器只会执行一次的特点

该单例模式的实现主要使用到了装饰器、闭包等概念。

闭包: 在函数singlton中，cls和instances就是充分利用了闭包的特点。在退出函数singleton，函数生命周期结束的时候，cls和instances依然存在，因为他们被内部函数引用了，所以不会被回收。关于闭包的讲解可以看下这篇文章：

装饰器：上面使用装饰器实现的单例模式是上就是和调用函数实现单利模式一样的。使用装饰器就是一个语法糖，看起来更Pythonic。关于装饰器可以看下这篇文章：装饰器

通过装饰之后，实际上每次调用Counter()实际上就在执行_singleton()函数，这个函数是个闭包，所以就使用了依然存在的变量cls和instances，通过_singleton()函数的内部逻辑，就实现了一个类只创建一个对象，即单例模式。

方法二：继承基类，__new__类内置方法实现单例

实际上利用的就是类变量对所有对象都是唯一的,即cls._instance，即通过__new__创建对象的时候，永远只创建一个。

该方法实现的单例模式主要就是使用了类的内置方法__new__，该方法是类创建对象时最先调用的方法，这个方法就是创建一个类的实例，其他内置类方法参数中的self实际上就是这个__new__创建的实例。类的内置方法__init__是在创建实例对象之后调用，初始化对象的一些属性。

共享模式实现的单例模式，实际上是每一次创建对象，是一个新的对象，但是新对象和之前的对象共享相同的属性，或者说共享状态。
